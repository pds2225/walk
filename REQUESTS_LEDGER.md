## 10. [신규 요청 설계] walk 자가개선 — "혼자 가상환경 테스트 + 실사용 정오답 학습"

사용자 요청(2026-07-16): *"walk를 너 혼자 가상환경에서 테스트해 UX 불편을 개발에 반영·지속보완 / walk 세션에서 '이 추천이 맞았나 틀렸나' 실사용 판단을 학습하길 원함."*
→ 지난 답변에서 mail로 잘못 돌렸음. **대상은 walk로 확정.** 두 루프로 분해:

### A. 자율 UX 테스트 루프 (dev-side, 안전 — 가상환경)
현 자산 재사용: `scenarios.py`(시뮬 GPS 경로)·`run_demo.py`·AppTest·헤드리스 부팅·과거 720회 걷기 시뮬(scratchpad). night-autodev에 **UX 지표 하네스**를 얹는 구조.
- 다수 시뮬 보행을 엔진+렌더에 태워 **UX 지표** 측정: 재탐색 빈도·회전예고 리드타임·마커 지터·상태 플립플롭·도착 판정 lag.
- 임계 초과 = 'UX 불편'으로 플래그 → 최소 수정+회귀테스트 → 야간 반복.
- 성격: 시뮬만 쓰므로 안전. night-autodev 확장으로 구현 가능(신규 스킬 불필요).

### B. 실사용 정오답 학습 루프 (product-side, UI 추가 필요)
- **피드백 캡처**: 안내 이벤트(재탐색·회전예고·도착) 뒤 가벼운 **👍맞음/👎틀림** 컨트롤 → {이벤트·상황·판정}을 로그(localStorage 또는 파일/시트)에 축적.
- **학습·반영**: 축적된 정오답에서 패턴 색출(예: "이 상황의 재탐색 판정이 자주 틀림") → 임계·로직 튜닝안 도출 → PR/야간 반복.
- 성격: 앱 UI에 피드백 위젯 추가 = **제품 방향 변경** → 착수 전 범위 확인 권장.
- 전제: "맞음/틀림"을 누가 라벨? = 사용자가 실사용 중 한 번씩 표시 → 그게 정답지.

*(상세 답변·판단은 세션 채팅 기록. 본 §은 원장 추적용 요약. 착수는 §10 확인 후.)*
