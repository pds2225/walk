## 0. 원(原) 요구사항 (SSOT — PROMPT.md/PLAN.md/DONE.md)

- **무엇:** Walk(Milestone 1) — **보행 경로 이탈 감지 엔진**(Pedestrian Route Deviation Detection Engine).
- **핵심 문제:** 보행 내비에서 ①회전 지점을 직진 통과 ②골목 잘못 들어 조금씩 이탈 ③한참 뒤에야 재탐색 ④GPS 한 번 튀어도 즉시 오탐 — 을 **조기·정확**하게 감지.
- **해결:** 연속 위치 샘플(위도·경도·방향·속도·타임스탬프)을 경로(RouteModel)와 대조해 `on_route / drifting / deviated / passed_turn` 4상태를 즉시 반환.
- **구조:** 모노레포 — `packages/route-engine`(핵심 엔진), `streamlit_walk_engine`(Streamlit UI/실증).
- **범위 규칙:** PROMPT.md 범위 밖 기능 금지, 프로젝트명 `walk` 유지, `1_Navigation.py` 최소 변경, `.env`·workflows 임의 수정 금지, 완료 판단은 DONE.md.
- **소셜 비전(2026-07-13 확정, 방향성):** '걸은 길+사진 경험 공유' 소셜 — 경험이 다음 사람의 길안내 데이터가 되는 선순환. 정본 `VISION_SOCIAL.md`. (1~2단계 현 스택 가능, 3단계 피드는 백엔드 필요 — **미착수**)

## 8. [신규 요청] 자동개발 방식 — "메일 정오답 학습 → 자가학습 → 개발 반영 지속보완"

사용자 요청(2026-07-16): *"자동개발하면 어떤 식으로 기능개선이 진행되나? 내가 원하는 방식 = 기존 메일을 보고 맞는 것/틀린 것을 학습해서 자가학습하고 그걸 개발에 반영해 지속 보완."*

이 요청은 walk가 아니라 **mail 프로젝트(D:\mail, 정부지원사업 메일 모니터링)** 대상 자가학습 루프로 판단됨(맥락에 대화형 답변 §은 채팅 참조). 현재 보유 자산으로 대부분 구현 가능:
- 이미 존재: `mail-accuracy-orchestrator`(FP/FN 측정 S0~S5 파이프라인), `agent-self-learning`(세션 평가·교훈 추출), `night-autodev`(야간 무인 개선).
- 사용자가 원하는 루프 = **① 발송된 추천 메일의 맞음(적합)/틀림(오탐·누락)을 라벨로 축적 → ② 그 라벨로 판정 로직의 빈틈을 자동 색출 → ③ 최소 수정+회귀테스트로 개발 반영 → ④ 야간 반복(지속 보완)**.
- 다음 액션(사용자 확인 후): mail 레포에서 이 4단계를 night-autodev에 연결하는 배선 설계(신규 스킬 생성 전 중복 확인 — 위 3개 자산 재사용).

*(상세 답변·판단은 세션 채팅에 기록. 이 항목은 원장 추적용 요약.)*
